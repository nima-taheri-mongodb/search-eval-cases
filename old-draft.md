## Appendix — Common Alternative Patterns

| Decision Point | Option A | Option B | When to Choose |
| :---- | :---- | :---- | :---- |
| **Genre filtering** | `text` operator on `genres` | `equals` operator on `genres` | `equals` for exact match; `text` if tokenization helps |
| **Date filtering** | `range` on `year` (int) | `range` on `released` (date) | `year` is simpler; `released` is more precise |
| **Search path** | `"path": "plot"` | `"path": [ "plot", "fullplot" ]` | Single field for speed; multi for recall |
| **Vector pre vs post filter** | `filter` in `$vectorSearch` | `$match` after `$vectorSearch` | Pre-filter if field is in vector index; post-filter otherwise |
| **Vector limit for post-filter** | `"limit": 10` | `"limit": 50` (inflated) | Inflate when post-filter will reduce results |
| **Embedding model** | `voyage-3-large` | `voyage-3.5` / `voyage-3.5-lite` | Must match index embedding model |
| **Hybrid search** | `$rankFusion` | `$scoreFusion` | RRF for rank-based fusion; score fusion for score-based |
| **Score normalization** | `sigmoid` | `minMaxScaler` | Sigmoid for smooth curves; minMax for linear scaling |
| **Pagination** | `$skip` \+ `$limit` | `searchAfter` / `searchBefore` | Offset for shallow pages; cursor for deep pagination |
| **Sorting** | Native `sort` in `$search` | Post-search `$sort` | Native is faster; post-search for non-indexed fields |
| **Count** | `$count` stage | `$searchMeta` count | `$searchMeta` is more efficient (no doc fetch) |
| **Facets** | `$searchMeta` facets | `$unwind` \+ `$group` | `$searchMeta` requires facet index mapping; manual group works without |
